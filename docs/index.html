What is RDTR?

RDTR is a deterministic, WASM-powered runtime for building remote terminal user interfaces. It replaces HTML, CSS, and JavaScript with:

  • Declarative layout trees
  • Constraint-based layout engine
  • Overlay stack system
  • Sandboxed WASM execution
  • Binary protocol over QUIC

Why RDTR?

Browsers are too heavy for terminal-native interfaces. Traditional TUIs are imperative and stateful. RDTR provides:

  • Deterministic rendering
  • Validated mutation model
  • Secure execution boundary
  • Minimal surface area
  • Keyboard-first UX

How It Works

BaseLayoutTree + OverlayStack + TerminalSize = Deterministic Frame WASM → MutationOps → Engine → Layout → Diff → Terminal

Specification

RDTR is defined as a formal specification including:

  • Binary encoding standard
  • Layout resolution algorithm
  • Overlay positioning model
  • WASM host API
  • Compliance test suite
